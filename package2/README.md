## 💡 작업 내용

- step1 main에서 유효성을 검증하였다.
- step2 객체 내부에서 유효성을 검증하였다. 이를 위해 Lion클래스에서 사용자 입력을 검증하기 위해 check메서드를 구성해주었다.
- step3 접근 제어자에 따른 필드 접근 테스트를 위해 name필드에 직접 값을 대입하여 출력값을 확인하고 default, private접근제어자를 사용한 경우에는 오류가 나는것을 확인하였다.


## 🤔 느낀 점 / 어려웠던 점
- 전체적으로 문제를 이해하는것부터 어려웠던 것 같다. 객체지향(데이터와 기능을 하나로 묶는다)에 대한 개념과 클래스, 객체, 메서드에 대해 다시 한번 정리해서 알게 되었고
private, default, public 접근제어자들에 대해 새롭게 알게 되었다. private의 경우 클래스 안에서만 변수를 쓸 수 있게 함으로써 클래스 밖에서 마음대로 값을 건드리는걸
막아서 객체가 원래 가지고 있던 규칙에서 벗어나는걸 방지한다는걸 깨닫게 되었다. 그 외 public은 어디서든 접근 가능하며 default는 접근제어자를 쓰지 않은 상태이며 다른 패키지에서만 쓸 수 없다는 것을
알게 되었다.
